* Spring aop advice type
- @Around: join point running, return, catch, finally
  - most strong advice type. can all things
    - joinPoint.proceed() can invoke next advice or target method
      - multiple invoke joinPoint.proceed() for retry
    - change returning value
    - catch exception
    - use finally
  - first parameter must be ProceedingJoinPoint
- @Before: before join point running 
  - can't control target method running
  - auto invoke target after advice return  
- @After Returning: after join point running
  - returning value must be same as parameter name
  - parameter type must be assignable to return type
  - can't change returning value. but can manipulate target method return value 
- @After Throwing: after join point throwing exception
  - thrwing value must be same as parameter name
  - parameter type must be assignable to throwing type
- @After: after join point returning or throwing exception(finally)
  - similar to finally
  - normally be used for resource release

* JoinPoint interface - Advice can use first parameter(except @Around)
- getArgs(): arguments
- getThis(): proxy object
- getTarget(): target object
- getSignature(): method signature
- toString(): advice info

*ProceedingJoinPoint interface*
- proceed(): invoke next advice or target method

* Advice type order*
- @Around > @Before > @After > @After Returning > @After Throwing
- runtime order: @Around > @Before > @After Returning > @After Throwing > @After > @Around(after returning)

* caution*
- use @Around advice when really need
